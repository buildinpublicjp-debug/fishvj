## コンセプト

感情の方向を指定できるAI VJ。音量反応ではなく「会場をどんな感情状態にしたいか」で演出全体を導く。
世界観レイヤー: 漁港レイヴ「魚曼荼羅 / GYOGEN」— スマホを預け、くさやを焼き、その日獲れた魚の曼荼羅を100人で浴びる。
締めのコピー: 「このソフトが生成するのは映像ではありません。未来の食卓です」

## アーキテクチャ（2層）

### 生成レイヤー（遅くてよい / OpenAI画像生成API）
- イベント中、裏で常時パーツを生成し続ける
  - 魚の個体スプライト（`gpt-image-1.5`、透明背景、魚種いろいろ）
  - 曼荼羅リング紋様（`gpt-image-2`、プロジェクタのアスペクト比）
  - 背景テクスチャ（`gpt-image-2`、深海 / 星空、プロジェクタのアスペクト比）
- 生成ルーティングは素材種別で固定する。`gpt-image-2` の解像度はAPI制約内でプロジェクタ比率に合わせる
- 生成物は「クリップ棚」（Resolume風サムネイルグリッド）に自動追加 → クリックでレイヤーに差し込み
- **棚が自動増殖するResolume = 既存VJソフトに存在しない画。OpenAI審査への直球回答**
- UI上に生成キュー表示（「新しい魚が生まれています… 3/∞」）

### 駆動レイヤー（60fps / WebGL shader、ローカル描画）
- LLM/生成APIは毎フレーム関与しない。演出設計を作るのはAI、描画はローカル

## レイヤースタック（Resolume型）

1. Layer 1（奥）: 背景シェーダー（深海/星空/ノイズ）
2. Layer 2（中）: 2.5D魚群パーティクル（奥行き配置、音駆動、curl noise）
3. Layer 3（前）: 曼荼羅リング（生成画像を放射状に敷き詰め）
4. Layer 4（最前）: 生成画像ポップ + 条件付きヒーロー魚GLB
- 各レイヤー: opacity + blend mode (add / screen)
- グローバルポストFX: RGBシフト / ブラー / 万華鏡(kaleidoscope) / フィードバック
  - 万華鏡は実装極小で「曼荼羅化」できる飛び道具
- ヒーロー魚GLBは事前完成・ブラウザ読込確認済みのseedだけを使う。未完成なら関連機能を無条件dropし、ライブimage-to-3Dは行わない

## 音声解析

### 入力2系統
1. **本線: mp3直接再生** → AudioContextに直結（解析クリーン、事故ゼロ）
2. **マイクモード**: getUserMedia
   - HTTPS必須（Vercel等でクリア）
   - AudioContextはユーザータップ後に起動 → 「START RITUAL」ボタンを儀式の入口として演出化
   - **必須設定**: `{ echoCancellation: false, noiseSuppression: false, autoGainControl: false }`（デフォルトだと音楽がノイズ扱いで潰される）
   - iPhoneでも同URLで動作（質疑用の保険。本線はMac + 会場スクリーン）

### 帯域分解（AnalyserNode, fftSize 2048、5帯域）
- kick: 30-100Hz
- bass: 100-250Hz
- mid: 250Hz-2kHz
- high: 2k-6kHz
- hihat: 6k-12kHz（独立帯域）

### 楽器・声の検出
| 対象 | 検出条件 | 演出マッピング |
|---|---|---|
| キック | 30-100Hz 急峻な立ち上がり | 魚群の脈動（全体のハートビート）+ BPM推定の基準 |
| スネア/クラップ | 150-250Hz と 1-4kHz の同時バースト | 曼荼羅リング反転フラッシュ（裏打ちの気持ちよさ） |
| ハイハット | 独立hihat帯6-12kHzの短トランジェント (<50ms) | 鱗の発光スパークル |
| vocal/lead proxy | 200Hz-3kHz 持続エネルギー（数百ms、変動なめらか） | 中央曼荼羅の開花 + ゴッドレイ強化（チャント=神聖演出） |

- `vocal/lead proxy` は汎用的な歌声分離ではなく、中域の持続エネルギーを使うデモ曲向けヒューリスティック

### BPM同期（音量反応ではなく拍位相駆動）
- キックオンセット間隔の中央値 → BPM推定（70-180クランプ、倍/半分補正）
- 本線mp3は既知BPM（予定138）をメタデータとして保持。推定安定前または信頼度低下時は既知値で位相駆動
- ビート位相 0-1 を常時算出
- 毎拍: 魚群脈動（位相0で収縮→リリース）
- 4拍: 曼荼羅リング1段回転
- 16拍(4小節): ブレンド切替 + 棚から新素材投入
- ビルドアップ検出（エネルギー持続上昇）: 渦収縮 + 彩度上昇
- ドロップ検出（急落→急上昇）: 万華鏡全開 + 曼荼羅開花
- UI: BPM表示 + タップテンポ

## 3モード（マクロプリセット、キー1/2/3、遷移2秒lerp）

### MYSTIC
- 色相 220-260（藍〜紫）/ 彩度中 / 魚速度 0.3x
- 曼荼羅回転ゆっくり / 万華鏡6分割 / blend: screen
- カメラ: ゆっくり沈降（深海へ降りる）
- 生成プロンプト: "deep sea, mandala, bioluminescent fish, indigo"

### SENSUAL
- 色相 0-40（赤〜橙）/ 彩度高め低照度
- 魚の動き流体的（curl noise強）/ opacityがBPMの1/2周期で呼吸
- 万華鏡8分割
- カメラ: 固定 `sin/cos` による緩いうねり旋回
- 生成プロンプト: "warm amber, flowing silk, fish scales glowing"

### EUPHORIC
- 色相フルレンジ回転 / 彩度最大 / 魚速度 1.5x
- ドロップでパーティクル爆発 / 万華鏡12分割 / blend: add
- カメラ: 急上昇 + 魚群中心へ突入 + 曼荼羅への吸引。ドロップで加速
- 生成プロンプト: "golden explosion, radial fish mandala, supernova"

- 3h版のカメラパスは固定数式のみ、実装10分timebox。`camera motion` OFFで即座に静止カメラへ戻せること
- 奥行き配置した魚Planeはカメラ視線追従。スプライン/汎用カメラシステムは対象外
- stretch goal: コア完成・本番URL・デモ動画確保後のみ「Describe the journey」を実装。GPT-5.6が自然言語を演出設計JSON（色・形・速度・展開）へ変換 + 生成プロンプトに反映

## 解像度

- 8K対応 = canvas解像度の問題（シェーダー描画なので生成画像を8Kにする必要なし）
- 素材1024pxでも数百体敷き詰めで密度勝負。曼荼羅=小パーツ反復なので相性良
- 当日初期値: 1920x1080 / devicePixelRatio上限1 / 魚2,000体
- 解像度スケールと魚数を実行時スライダーにし、会場マシンで60fpsを確認しながら上げる
- canvasはプロジェクタ実解像度を上限とする。4K/8Kは実測後に選択

## プレゼン構成（3分）

パネルネタ（〜75秒、1枚1言・12秒/枚）→ ライブデモ（90秒）→ 締め（30秒）

1. 【焚き火を囲む狩猟採集民】「人類の始まりです。火を囲み、獲物を食べ、仲間と生きていました」
2. 【神話画（天孫降臨/プロメテウス）】「世界中の神話が『天から降りてきた何か』を語っています。人類には仕込まれた使命があったのかもしれません」
3. 【AIイメージ】「それがこれです。我々はAIを産むために生まれてきた。……そして先月あたり、ほぼ完成しました」
4. 【宇宙】「AIはこれから宇宙を理解しにいきます。もう人間の仕事ではありません」
5. 【焚き火に戻る】「では、使命を終えた人間は何をするか。——始まりに還ります」
6. 【未来の食卓で魚を食べる男女】「これです。幸せそうですね」
7. 【焼き魚アップ】「魚を食べています」
8. 【漁港】「そこで漁港です」
9. 【スマホを籠に預ける】「入口でスマホを預けます。もう見れません」
10. 【くさや】「地元の方がくさやを焼いています。臭いです。でもそれがいい」
11. 【魚曼荼羅（生成済コンセプトアート1枚目）】「夜、その日獲れた魚が曼荼羅になって壁一面に増殖します」
12. 【プロダクトイメージ（2枚目: 3オーブUI）】「……というイベントを支えるAI VJを、今日作りました」
→ 実デモ（UIが12枚目と同一 = 「本当に作った」の証明）
→ 締め【食卓写真に戻る】「このソフトが生成しているのは映像じゃありません。未来の食卓です」

- 表現注意: 「人間を持っていく」ではなく「演者と観客が行きたい感情状態を一緒に選ぶ」
